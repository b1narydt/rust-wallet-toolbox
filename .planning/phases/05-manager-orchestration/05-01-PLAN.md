    **Implementation of `set_active` on WalletStorageManager** — add the method to the existing impl block in manager.rs, following the 8-step TS algorithm from 05-RESEARCH.md:

    ```rust
    pub async fn set_active(
        &self,
        storage_identity_key: &str,
        prog_log: Option<&(dyn Fn(&str) -> String + Send + Sync)>,
    ) -> WalletResult<String>
    ```

    **Step 0 — Auto-init:** If `!self.is_available()`, call `self.make_available().await?`.

    **Step 1 — Validate target store:** Lock state, iterate stores, find index whose `settings.storage_identity_key == storage_identity_key`. Return `WalletError::InvalidParameter { parameter: "storage_identity_key", must_be: "registered with this WalletStorageManager..." }` if not found. Release state lock.

    **Step 2 — Early return:** If `storage_identity_key == self.get_active_store().await?` AND `self.is_active_enabled().await`, return `Ok("... unchanged\n".to_string())`.

    **Step 3 — Acquire sync lock:** `let _guards = self.acquire_sync().await?;`

    **Step 4 — Conflict resolution:** Lock state. If `!state.conflicting_active_indices.is_empty()`:
    - Build conflict source list: all conflicting_active_indices + active_index, minus new_active_index.
    - Clone Arc refs and settings for new_active and each conflict source. Drop state lock.
    - For each conflict source: `self.sync_to_writer(new_active_arc.as_ref(), conflict_arc.as_ref(), &new_active_settings, &conflict_settings, prog_log).await?`

    **CRITICAL: Do NOT hold state mutex across sync_to_writer calls.** Clone Arcs and settings first, drop the lock, then do async I/O. Follow the pattern from update_backups (manager.rs:1536).

    **Step 5 — Determine backup_source:** If conflicts existed, backup_source = new_active; else backup_source = current_active. Clone Arc + user_id before releasing state.

    **Step 6 — Provider-level set_active:** Build AuthId with `identity_key` and `backup_source_user_id`, call `backup_source_storage.set_active(&auth, storage_identity_key).await?`.

    **Step 7 — Propagate to all other stores:** Lock state, update each store's cached `user.active_storage = storage_identity_key`. Collect list of (store_arc, store_settings) for all stores whose sik != backup_source_sik. Drop state lock. For each: `self.sync_to_writer(store_arc.as_ref(), backup_source_arc.as_ref(), &store_settings, &backup_source_settings, prog_log).await?`.

    **Step 8 — Re-partition:** `self.is_available_flag.store(false, Ordering::Release); self.do_make_available().await?;` Use `do_make_available()`, NOT `make_available()` — the latter would deadlock because acquire_sync already holds reader_lock.

    Return `Ok(log)` with accumulated prog_log messages.

Each test creates a WalletStorageManager with in-memory SQLite stores, calls set_active, and verifies the expected outcome.

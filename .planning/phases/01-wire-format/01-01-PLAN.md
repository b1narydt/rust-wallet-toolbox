       - Split the single `FORMAT` constant into two: `PARSE_FORMAT = "%Y-%m-%dT%H:%M:%S%.f"` (unchanged, tolerant) and `SERIALIZE_FORMAT = "%Y-%m-%dT%H:%M:%S%.3f"` (exactly 3 ms digits).

       - In the `option` submodule: apply the SAME split. `option::serialize` must use `format!("{}Z", d.format(SERIALIZE_FORMAT))` for `Some(d)`. `option::deserialize` uses `PARSE_FORMAT`.

    2. **Fix `src/storage/sync/sync_map.rs`:**
       - Add `#[serde(skip_serializing_if = "Option::is_none")]` to ALL `Option<...>` fields in the `SyncChunk` struct: `user`, `proven_txs`, `output_baskets`, `transactions`, `outputs`, `tx_labels`, `tx_label_maps`, `output_tags`, `output_tag_maps`, `certificates`, `certificate_fields`, `commissions`, `proven_tx_reqs` (13 fields total).
       - Do NOT touch `EntitySyncMap` or `SyncMap` -- they have no optional fields that need this.

    Run `cargo test --test table_tests` to confirm all tests pass (GREEN phase).

    IMPORTANT anti-patterns to avoid:
    - Do NOT change `PARSE_FORMAT` to `%.3f` -- that breaks parsing of variable-precision timestamps.
    - Do NOT add `skip_serializing_if` to table struct `Option<T>` fields (like `Option<i64>` on Output) -- those correctly serialize as null, matching TS.

Replace every `todo!()` stub in the `impl WalletStorageProvider for StorageClient<W>` block with a one-liner call to `self.rpc_call(wire_name, params)`.

Each method follows this pattern:
```rust
async fn method_name(&self, auth: &AuthId, args: &ArgType) -> WalletResult<ReturnType> {
    self.rpc_call("wireMethodName", vec![
        serde_json::to_value(auth)?,
        serde_json::to_value(args)?,
    ]).await
}
```

**CRITICAL wire name rules (from research):**
- `find_output_baskets_auth` -> wire name `"findOutputBaskets"` (NO Auth suffix). This is the only method that drops Auth from the wire name.
- `internalize_action` -> wire name `"internalizeAction"`, params `[auth, args]`. IGNORE the `services: &dyn WalletServices` parameter entirely (it is not serializable and has no TS equivalent). Accept it in the signature but do not serialize it.
- `migrate` -> params `[storage_name, storage_identity_key]` (positional order matters).
- `find_or_insert_sync_state_auth` -> params `[auth, storage_identity_key, storage_name]` (3 params, not a struct).
- `set_active` -> params `[auth, new_active_storage_identity_key]` (string, not struct).
- `find_or_insert_user` -> params `[identity_key]` (string, not struct).
- Methods with no params: `destroy` -> `[]`

For `&str` parameters that need to be serialized into the params array, use `serde_json::Value::String(param.to_string())`.

For `&AuthId` and struct reference params, use `serde_json::to_value(param)?`.

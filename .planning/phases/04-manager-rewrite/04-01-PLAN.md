  <action>
    Replace the entire `src/storage/manager.rs` file. Remove all old StorageProvider/StorageReader/StorageReaderWriter trait impls and the write_with_backup macro. The new file contains:

    1. **ManagedStorage struct** (private):
       - `storage: Arc<dyn WalletStorageProvider>`
       - `is_storage_provider: bool` (set from `storage.is_storage_provider()` at construction)
       - `is_available: AtomicBool` (starts false)
       - `settings: tokio::sync::Mutex<Option<Settings>>`
       - `user: tokio::sync::Mutex<Option<User>>`
       - Constructor `fn new(storage: Arc<dyn WalletStorageProvider>) -> Self`

    2. **ManagerState struct** (private, guards partition state):
       - `stores: Vec<ManagedStorage>` — all providers
       - `active_index: Option<usize>` — index of active store in `stores`
       - `backup_indices: Vec<usize>` — indices of backup stores
       - `conflicting_active_indices: Vec<usize>` — indices of conflicting stores

    3. **WalletStorageManager struct** (public):
       - `state: tokio::sync::Mutex<ManagerState>` — partition state
       - `identity_key: String`
       - `reader_lock: tokio::sync::Mutex<()>`
       - `writer_lock: tokio::sync::Mutex<()>`
       - `sync_lock: tokio::sync::Mutex<()>`
       - `sp_lock: tokio::sync::Mutex<()>`
       - `is_available: AtomicBool`
       - `services: tokio::sync::Mutex<Option<Arc<dyn WalletServices>>>`

    4. **Constructor** `pub fn new(identity_key: String, active: Option<Arc<dyn WalletStorageProvider>>, backups: Vec<Arc<dyn WalletStorageProvider>>) -> Self`:
       - Create ManagedStorage for active (if Some) and each backup
       - Build initial stores vec: active first (if present), then backups
       - Initialize ManagerState with no partition yet (active_index: None, etc.)
       - All four locks initialized as `tokio::sync::Mutex::new(())`

    5. **Accessor methods** (zero-lock getters matching TS property accessors):
       - `pub fn get_user_identity_key(&self) -> &str`
       - `pub fn auth_id(&self) -> &str` (alias, same as get_user_identity_key)
       - `pub fn is_available(&self) -> bool` — reads AtomicBool with Acquire ordering
       - `async fn get_active(&self) -> WalletResult<Arc<dyn WalletStorageProvider>>` — lock state, return stores[active_index].storage.clone() or error if not available
       - `async fn get_auth(&self, must_be_active: bool) -> WalletResult<AuthId>` — build AuthId from identity_key + cached user_id + is_active from partition state. If `must_be_active` and not `is_active_enabled`, return `WERR_NOT_ACTIVE` error.
       - `pub async fn is_active_enabled(&self) -> bool` — computed: `active.settings.storage_identity_key == active.user.active_storage AND conflicting_active_indices.is_empty()`. Lock state to read.
       - `pub async fn can_make_available(&self) -> bool` — returns true if stores is non-empty (locks state)
       - `pub async fn get_active_settings(&self) -> WalletResult<Settings>` — lock state, return clone of active store's cached settings (error if not available)
       - `pub async fn get_active_user(&self) -> WalletResult<User>` — lock state, return clone of active store's cached user (error if not available)
       - `pub async fn get_active_store(&self) -> WalletResult<String>` — returns active's `settings.storage_identity_key`
       - `pub async fn get_active_store_name(&self) -> WalletResult<String>` — returns active's `settings.storage_name`
       - `pub async fn get_backup_stores(&self) -> Vec<String>` — returns storage_identity_keys of backup stores
       - `pub async fn get_conflicting_stores(&self) -> Vec<String>` — returns storage_identity_keys of conflicting stores
       - `pub async fn get_all_stores(&self) -> Vec<String>` — returns all store storage_identity_keys
       - `pub async fn is_active_storage_provider(&self) -> WalletResult<bool>` — returns active store's `is_storage_provider` flag
       - `pub fn is_storage_provider(&self) -> bool` — always returns `false` (the manager itself is NOT a storage provider, matching TS behavior)
       - `pub async fn get_user_id(&self) -> WalletResult<i64>` — returns `self.get_auth(false).await?.user_id.ok_or(WERR_INVALID_OPERATION)`
       - `pub async fn get_settings(&self) -> WalletResult<Settings>` — alias for `get_active_settings()` (TS parity name)
       - `fn verify_active(&self, state: &ManagerState) -> WalletResult<usize>` — private helper: returns active_index or `WERR_INVALID_OPERATION` if active_index is None. Used internally to DRY-check active availability.

    6. **makeAvailable** `pub async fn make_available(&self) -> WalletResult<Settings>`:
       - Acquire reader_lock for idempotency (per research Pitfall 3)
       - If already available, return cached active settings
       - Lock state mutex
       - For each store in stores: call `store.storage.make_available().await`, cache settings; call `store.storage.find_or_insert_user(&self.identity_key).await`, cache user (note: the trait method is `find_or_insert_user(&str)`, NOT `find_or_insert_user_auth`); set store.is_available = true
       - Partition logic (direct TS translation):
         a. First store is default active (active_index = 0)
         b. For each subsequent store: if store.user.active_storage == store.settings.storage_identity_key AND current active is NOT enabled-active → swap: old active becomes backup, this store becomes active
         c. Otherwise push index to backup_indices
         d. Second pass: for each backup, if backup.user.active_storage != active.settings.storage_identity_key → move to conflicting_active_indices
       - is_active_enabled: `active.settings.storage_identity_key == active.user.active_storage AND conflicting_active_indices.is_empty()`
       - Set self.is_available = true (AtomicBool with Release)
       - Return active settings clone

    7. **Lock helpers** (private async methods, NOT closure-based per research Pitfall 6):
       - `async fn acquire_reader(&self) -> tokio::sync::MutexGuard<'_, ()>` — acquires reader_lock. **Auto-init**: if `!self.is_available()`, call `self.make_available().await?` first (matches TS `getActiveLock` behavior which calls `makeAvailable()` if not yet available).
       - `async fn acquire_writer(&self) -> (tokio::sync::MutexGuard<'_, ()>, tokio::sync::MutexGuard<'_, ()>)` — acquires reader_lock then writer_lock, returns both guards. Same auto-init check.
       - `async fn acquire_sync(&self) -> (MutexGuard, MutexGuard, MutexGuard)` — reader + writer + sync. Same auto-init check.
       - `async fn acquire_storage_provider(&self) -> (MutexGuard, MutexGuard, MutexGuard, MutexGuard)` — all four. Same auto-init check.

       The auto-init check is safe because `make_available()` is idempotent (guarded by `is_available` AtomicBool). This matches the TS pattern where every `getActiveLock()` call triggers `makeAvailable()` if needed.

       The research explicitly recommends AGAINST closure-based run_as_* (Pitfall 6) due to lifetime issues with dyn trait objects. Instead, callers acquire guards, do work, and guards drop automatically.

    8. **Rewrite tests/manager_tests.rs**: Remove all old tests. Create new test module using `Arc<dyn WalletStorageProvider>` (SqliteStorage satisfies this via blanket impl). Tests:
       - `test_constructor`: new with identity_key + active + 2 backups, verify stores count
       - `test_managed_storage_cache`: verify settings/user are None before makeAvailable
       - `test_make_available_single_store`: 1 store → becomes active, is_available = true
       - `test_make_available_partition`: 2 stores → active + 1 backup
       - `test_hierarchical_locks`: verify acquire_reader/writer/sync/sp don't deadlock when used correctly
       - `test_getter_methods`: after makeAvailable, verify is_active_enabled, get_active_settings, get_active_user, get_active_store, get_backup_stores, get_all_stores return correct values
       - `test_verify_active_before_available`: calling get_auth(true) before makeAvailable returns WERR_NOT_ACTIVE
       - `test_auto_init_in_lock_helpers`: acquire_reader on a not-yet-available manager triggers makeAvailable automatically
       - `test_make_available_zero_stores`: new(identity_key, None, vec![]) then makeAvailable returns WERR_INVALID_PARAMETER
       - `test_make_available_conflicting_actives`: 3 stores with differing user.active_storage — verify conflicting_active_indices populated correctly
       - `test_make_available_swap`: second store has user.active_storage matching its own settings AND first store is NOT enabled-active — second becomes active
       - `test_make_available_idempotent`: call makeAvailable twice, second call returns immediately (verify via settings cache hit, no second find_or_insert_user call)

    CRITICAL NOTES:
    - Remove ALL old trait impls (StorageReader, StorageReaderWriter, StorageProvider for WalletStorageManager). These will NOT be re-added — the new manager uses inherent methods with WalletStorageProvider, not the old trait hierarchy.
    - The old `write_with_backup` macro is deleted. New pattern: writes go to active only, backups sync via updateBackups (Plan 02).
    - Use `tokio::sync::Mutex` everywhere (never std::sync::Mutex) per accumulated decisions.
    - Keep `use` imports minimal — only import what's needed for this task. Sync-related imports come in Plan 02.
    - The Wallet struct in `src/wallet/` holds `WalletStorageManager` directly (not via trait) — no trait impl needed on the manager.

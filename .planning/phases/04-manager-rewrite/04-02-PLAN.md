    - Test: sync_to_writer loops getSyncChunk/processSyncChunk until done=true, accumulates inserts+updates
    - Test: sync_from_reader overrides chunk.user.active_storage with writer's cached value before calling processSyncChunk (Pitfall 5)
    - Test: update_backups iterates all backup stores calling sync_to_writer, one backup failure does not block others
  </behavior>
  <action>
    Add to `src/storage/manager.rs`:

    1. **`make_request_sync_chunk_args` free function**:
       - Parse `sync_state.sync_map` JSON string into `SyncMap` via `serde_json::from_str`
       - Build `Vec<SyncChunkOffset>` from each EntitySyncMap: name = entity_name, offset = count (the `count` field tracks cumulative items received in the current sync window — this IS the pagination offset, matching TS `ess.count`)
       - Return `RequestSyncChunkArgs { from_storage_identity_key: sync_state.storage_identity_key, to_storage_identity_key, identity_key, since: sync_state.when, max_rough_size: 10_000_000, max_items: 1000, offsets }`

    2. **`sync_to_writer` async method** on WalletStorageManager:
       - Signature: `async fn sync_to_writer(&self, writer: &dyn WalletStorageProvider, reader: &dyn WalletStorageProvider, writer_settings: &Settings, reader_settings: &Settings, prog_log: Option<&dyn Fn(&str) -> String>) -> WalletResult<(i64, i64, String)>`
       - `prog_log` is an optional progress callback matching TS `progLog?: (s: string) => string`. If None, use identity (no logging). Returns accumulated log string along with inserts/updates.
       - Must be called while sync lock is held (caller's responsibility)
       - Build AuthId from self.identity_key
       - Loop:
         a. `let (ss, _) = writer.find_or_insert_sync_state_auth(&auth, &reader_settings.storage_identity_key, &reader_settings.storage_name).await?`
         b. `let args = make_request_sync_chunk_args(&ss, &self.identity_key, &writer_settings.storage_identity_key)`
         c. `let chunk = reader.get_sync_chunk(&args).await?`
         d. `let r = writer.process_sync_chunk(&args, &chunk).await?`
         e. `inserts += r.inserts; updates += r.updates`
         f. If prog_log is Some, append `prog_log(&format!("chunk {} inserted {} updated {}", i, r.inserts, r.updates))` to log
         g. `if r.done { break; }`
       - Return (inserts, updates, log)

    3. **`sync_from_reader` async method** on WalletStorageManager:
       - Signature: `async fn sync_from_reader(&self, writer: &dyn WalletStorageProvider, reader: &dyn WalletStorageProvider, writer_settings: &Settings, reader_settings: &Settings, prog_log: Option<&dyn Fn(&str) -> String>) -> WalletResult<(i64, i64, String)>`
       - Same as sync_to_writer but with activeStorage override (Pitfall 5) and identity validation:
       - **Identity check**: Verify that reader's identity_key matches self.identity_key, otherwise return `WERR_UNAUTHORIZED` error (matches TS: `if (reader.identityKey !== this.identityKey) throw WERR_UNAUTHORIZED`)
       - After getting chunk from reader, before calling writer.process_sync_chunk, override `chunk.user.active_storage` with the writer's cached user.active_storage
       - This prevents the reader's active_storage from clobbering the writer's during sync

    4. **`update_backups` public async method**:
       - Signature: `pub async fn update_backups(&self, prog_log: Option<&dyn Fn(&str) -> String>) -> WalletResult<(i64, i64, String)>`
       - Acquire sync lock (reader + writer + sync)
       - Get active provider and its cached settings
       - Iterate backup_indices from state
       - For each backup: get its cached settings, call `sync_to_writer(backup, active, backup_settings, active_settings, prog_log)`
       - Catch per-backup errors with `warn!` logging, continue to next backup
       - Return aggregate (inserts, updates, log)

    5. **Add tests to manager_tests.rs**:
       - `test_make_request_sync_chunk_args`: construct a SyncState with known sync_map JSON, verify offsets and since are correct
       - `test_sync_to_writer`: use two SqliteStorage instances, insert data into reader, sync to writer, verify data appears
       - `test_update_backups`: manager with active + 2 backups, insert data to active, call update_backups, verify both backups have the data

    CRITICAL NOTES:
    - SyncChunk has `pub user: Option<User>` — the activeStorage override in sync_from_reader must handle the Option (if chunk.user is Some, override its active_storage)
    - The sync loop re-queries find_or_insert_sync_state_auth on every iteration because processSyncChunk updates the SyncState in the database (new sync_map, updated when)
    - Per Pitfall 4: the loop terminates because processSyncChunk advances the SyncState.when timestamp and SyncMap.count values, so the next iteration's RequestSyncChunkArgs asks for newer data

    Upgrade the stub delegation methods from Plan 01 Task 2 to use proper lock acquisition and AuthId-based dispatch. Also implement addWalletStorageProvider.

    1. **Upgrade delegation methods** — replace simple pass-throughs with locked versions:

       **Reader-level methods** (acquire reader lock):
       - `find_output_baskets_auth`, `find_certificates_auth`, `find_outputs_auth`, `find_proven_tx_reqs_auth`, `find_transactions_auth`
       - Pattern: `let _guard = self.acquire_reader().await; let active = self.get_active().await?; active.find_*_auth(&auth, args).await`

       **Writer-level methods** (acquire reader + writer locks):
       - `create_action_auth`, `process_action_auth`, `internalize_action_auth`
       - `abort_action_auth`, `insert_certificate_auth`, `relinquish_certificate`, `relinquish_output`
       - Pattern: `let (_rg, _wg) = self.acquire_writer().await; let active = self.get_active().await?; active.*_auth(&auth, args).await`

       **Sync-level methods** (acquire reader + writer + sync locks):
       - `update_backups` (already done in Task 1)
       - `set_active` (stub — full implementation in Phase 5/ORCH-01)

       **StorageProvider-level methods** (acquire all four locks):
       - `add_wallet_storage_provider`
       - `make_available` (already done in Plan 01)

    2. **get_auth helper** (if not already added in Plan 01):
       - `async fn get_auth(&self, must_be_active: bool) -> WalletResult<AuthId>`
       - Lock state, read active store's cached user
       - Build AuthId { identity_key: self.identity_key.clone(), user_id: user.map(|u| u.user_id), is_active: Some(must_be_active && is_active_enabled) }
       - If must_be_active and not is_active_enabled, return `WERR_NOT_ACTIVE` error

    2b. **Error codes** — ensure these TS-parity error conditions are implemented across all methods:
       - `WERR_INVALID_PARAMETER` — `makeAvailable()` when stores vec is empty
       - `WERR_INVALID_OPERATION` — `verify_active()` when active_index is None (not yet available)
       - `WERR_NOT_ACTIVE` — `get_auth(must_be_active=true)` when manager is not enabled-active
       - `WERR_UNAUTHORIZED` — `sync_from_reader()` when reader identity_key doesn't match manager's identity_key
       - `WERR_INTERNAL` — `find_or_insert_user` if returned userId differs from cached AuthId.user_id (consistency violation)
       Use existing `WalletError` variants or add new ones matching these TS error codes.

    3. **High-level methods** (list_actions, list_outputs, list_certificates):
       - These call into `storage::methods::*` modules that take `&dyn StorageReaderWriter`
       - The active provider via WalletStorageProvider doesn't expose StorageReaderWriter directly
       - Solution: For local providers (is_storage_provider() == true), downcast or use the storage::methods with the active provider's underlying StorageReaderWriter. For now, call the WalletStorageProvider-level equivalents (find_outputs_auth, etc.) which the blanket impl routes to the same code.
       - If the list_* methods need the raw StorageReaderWriter interface, keep them as-is using the active provider but via `self.get_active()` instead of `self.active`.

    4. **addWalletStorageProvider** `pub async fn add_wallet_storage_provider(&self, provider: Arc<dyn WalletStorageProvider>) -> WalletResult<()>`:
       - Acquire all four locks (storage_provider level)
       - Lock state
       - Create ManagedStorage for the new provider
       - Push to stores vec
       - Reset self.is_available to false
       - Drop state lock
       - Call self.make_available().await to re-partition
       - Return Ok(())

    5. **Add/update tests in manager_tests.rs**:
       - `test_delegation_methods`: create manager with active, call find_output_baskets (reader) and create_action (writer), verify they route to active
       - `test_add_provider_runtime`: create manager with 1 provider, add another via add_wallet_storage_provider, verify stores count increased and makeAvailable re-ran
       - `test_sync_prog_log`: sync_to_writer with a prog_log callback, verify log string contains chunk progress
       - `test_sync_from_reader_unauthorized`: sync_from_reader with mismatched identity_key returns WERR_UNAUTHORIZED
       - `test_find_or_insert_user_consistency`: if find_or_insert_user returns a different userId than cached, WERR_INTERNAL is raised

    6. **Ensure full compilation**: Run `cargo check --features sqlite` and fix any issues. The key concern is that removing the old StorageProvider/StorageReader/StorageReaderWriter trait impls may break call sites in src/wallet/ that used trait dispatch. Fix those to use inherent method calls.
